Ejercicio de clase 03formulario

Todos los formularios en web, manejan metodos para el envio de las diferentes peticiones que realiza el usuario. Debemos recordar que la mayoria de las apliucaciones en web trabajan sobre una arquitectura cliente servidor. Significa que el cliente realiza una peticion (request) y el servidor debe de atender ,ediente una respuesta (response).

Los metodos mas comunes son:

GET:

Este metodo se encarga de enviar la informacion del formulario por medio de la salida o url estandar del navegador, esto propicia que pueda existir inyecciones de codigfo realizando violaciones a la seguridad del sistema.

POST:

Este metodo ser encarga de enviar la infromacion ifgual por el metodo estandar; pero mediante un ocultamiento, esto significa que no es visible al usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo del sistema


  • GET solo se usa para cambiat la informacion de lectura.

  • POST sirve para cambiar la inforaion de un recurso por parte del servidor. (insertar registro)

  • Delete sirve para eliminar un recurso en especifico.

  • Put almacena la informaion de un recurso particular, para sua actuyalizacion.

  • Head regresa el codigo respuesta y cualquiera cabecera HTTP asociada.
